我们继续学习core graphic,我最终的目标是ios开发，所以从现在开始，我将只关注ios上的表现。

Core Graphics是除UIKit,foundatin之外最重要的库，因为UIKit依赖于Core Graphics，所以一般新建的IOS 项目都已经导入了Core Graphics库。在任何需要用到 Core Graphics的地方，用

Quartz是mac osx和ios用于界面描绘的底层API,它包含两部分Quartz2D（二维界面渲染）和Quartz Core（核心动画的库），CoreGraphics库已经包含了Quartz2D.

核心图形的几何坐标系统原点(origin(0,0))设在左上角，在这个坐标系统中，x轴坐标代表右移，y轴坐标代表下移。使用核心动画时，我们应该用点(points)来思考而不是用像素，因为像素会因为设备分辨率的不同而有所变化。比如，在iphone4的retain屏幕，和iphone3一样的屏幕大小，分辨率由480x320变成了960x640，他们的坐标系统仍然保持480x320不变。

Core Graphics工作在graphics context内，graphics context像一块画图，是你绘图操作的目标。在UIiew的drawrect內，ios会自动为你准备好graphics context，你需要做的就是用　UIGraphicsGetCurrentContext()　来获得当前的graphics context。 你在graphics context做的动作是顺序的，新的操作会基于上一次的操作。Core Graphics最常见的操作是重写UIView的drawrect方法，当ios需要重绘View时，它就会调用drawrect。这就表示这个方法会被频繁的调用，尤其在动画/缩放等操作的时候，所以这个方法必须尽量轻，作为开发者，你永远不应该自己调用这个方法，也不该在这里alloc或release对象。

当你需要重绘视图时，你可以用视图的setNeedsDisplay方法，它会告诉ios在合适的时候把这个视图重绘。

当你新建UIView的子类时，你会得到一个全空的graphics context，graphics context有很多的属性，就像其他的IOS对象一样。不像UIVIew，graphics context的属性时状态导向，意思是绘画动作都依赖于当前的属性。

当你使用graphics context绘画时，你会遇到如下的属性

  • Path
  • Stroke
  • Line
  • Width
  • Fill Color
  • Line Dash
  • Shadow
  • Clip Path
  • Blend Mode
  • Current Transform Matrix

Graphic Context Stack

另Core Graphic一个核心特性是graphics context堆栈，我们还是用画家和画布做类比，一个graphics context相当于一个平的画布，绘画动作在单独层上进行，它不允许你把内容分布到不同的层上。graphics context堆栈允许你保持当前的graphics context状态，当特定的动作发生后在还原回来。这对要求特定的状态或者或者临时记住当前动作的复杂绘画尤其有用。你可以在drawrect里嵌套多个保持和还原操作，而且你必须要还原graphics context。

当你和Core Graphics打交道的时候，大部分的时间你都会碰到paths, gradients, text,和images。所以下面就重点讲这四项!

PATH

path是最简单的绘画方式，就像小时候玩的游戏，在两点之间连线一样，path就是在多个点之间用直线或者Bezier曲线连接。有很多方法使用路径，其中最简单的方法是用UIBezierPath对象.

UIBezierPath用来定义几何路径，就像你之前看到的一样，当你定义了路径以后，你就可以对笔画和内容上色，或者检测点击时间，剪切当前路径。
