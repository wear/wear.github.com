coreImage

Apr 19th, 2012

本篇文字用于记录学习CoreImage

背景介绍

CoreImage是苹果的图像处理包，在OSX上已经存在一些年头了，从IOS５开始，苹果把它移到了iPhone上。CoreImage是Object-c框架，它包含了对基于像素的图片的处理和分析函数，包含各种滤镜，头像探测等。

使用CoreImage

使用CoreImage的步骤是

  1. 创建一个新的 CIImage
  2. 创建新的CIFilter，并提供足够的参数
  3. 生成输出图片，输出图片是一个新的CIImage实例，你可以用它渲染图片或者应用新的滤镜
  4. 渲染图片,渲染可以基于CPU或者　GPU

IOS上有三种表示图片的方法，分别是UIImage,CIImage和CGImageRef，从前缀可以看出他们的从属框架，他们底层使用的数据是一样的，但是使用了不同的封装以提供对象属性。下面的代码演示了如果创建一个CIImage，注意它是从CGImageRef创建的

CIImage和其他图片方法不一样的是它不是最终的产品，它像一个蓝图，你可以使用多个滤镜（滤镜链）来处理它，直到处理结束后在生成最终的结果。

Core Image Context

Core Image context用于使用各种滤镜后渲染一个CIImage，当新建一个图片上下文的时候，它会基于CPU或者GPU初始化，当用CPU初始化的时候，使用它就与使用你的程序run loop里的其他对象没什么区别，当使用GCD的时候，它会用openGL２.０来渲染图片。使用GPU渲染路径的好处是你当前的程序RunLopp丝毫不受影响。

CPU渲染了解和GPU渲染路径各有个的优缺点，使用CPU渲染滤镜的一个好处是你可以确保图片可以渲染完成，因为GPU渲染路径是不能转入后台运行的，GPU路径的话性能好。
