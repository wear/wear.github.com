Quartz变形

Quartz绘图模型定义了两种坐标，用户坐标和和设备坐标，用户坐标是一个浮点数，与设备分辨率和像素无关，当你想要打印你的文档时，Quartz会把用户坐标对应到设备坐标上。

你可以听过操作当前变形矩形(current transformation matrix,CTM)修改你的预定义的用户坐标。当你创建了图形会话后，CTM就是单元矩阵。

Quartz提供了５个函数让你可以方便的平移，缩放，旋转你的会话坐标，你可以定义操作的顺序，组合。Quartz同时也提供创建映射转换的函数，你可以先创建映射转换，它不会马上改变CTM,直到你决定使用时。使用Quartz函数你不需要理解内容的矩阵数据逻辑，只需要使用相应的内置函数就可以了。 注解：形象的说，就是操作当前的画布，移动画布，旋转画图，缩放画布。

播放本地音频

播放本地音频需要如下步骤

  1. Set up an AV Foundation Audio Session (AVAudioSession).
  2. Set up an AV Foundation Audio Player (AVAudioPlayer).
  3. Handle any interruptions.

AV Foundation Audio Session

AV Foundation Audio Session管理你的IOS设备上的音频体验，通过设置不同的属性，你就可以为当前用户提供定制话的体验。比如你的应用需要播放背景音乐，如果用户在进入App的时候已经在播放音乐，比如他正在听Ipod，这时候你就可以选择是用背景音乐覆盖IPod的音乐还是继续播放IPod音乐。

AV Foundation Audio Session是个单例类，意思是只有一个它的实例。当你获得了实例的应用之后，你的首要任务是设置它的代理与类别，代理决定如果相应中断，类别确定多个音频源时的操作。类别有两种主要区别，一种是可以后台播放的，新的音频会替换旧的。但是如果是Ambient类，Ambient会与当前音效混合，Solo Ambient会覆盖当前音效。

drawInG CuStOm uIVIewS

核心图形的几何坐标系统原点(origin(0,0))设在左上角，在这个坐标系统中，x轴坐标代表右移，y轴坐标代表下移。使用核心动画时，我们应该用点(points)来思考而不是用像素，因为像素会因为设备分辨率的不同而有所变化。比如，在iphone4的retain屏幕，和iphone3一样的屏幕大小，分辨率由480x320变成了960x640，他们的坐标系统仍然保持480x320不变。

Core Graphics工作在graphics context内，graphics context像一块画图，是你绘图操作的目标。在UIiew的drawrect內，ios会自动为你准备好graphics context，你需要做的就是用　UIGraphicsGetCurrentContext()　来获得当前的graphics context。 你在graphics context做的动作是顺序的，新的操作会基于上一次的操作。Core Graphics最常见的操作是重写UIView的drawrect方法，当ios需要重绘View时，它就会调用drawrect。这就表示这个方法会被频繁的调用，尤其在动画/缩放等操作的时候，所以这个方法必须尽量轻，作为开发者，你永远不应该自己调用这个方法，也不该在这里alloc或release对象。

当你需要重绘视图时，你可以用视图的setNeedsDisplay方法，它会告诉ios在合适的时候把这个视图重绘。

当你新建UIView的子类时，你会得到一个全空的graphics context，graphics context有很多的属性，就像其他的IOS对象一样。不像UIVIew，graphics context的属性时状态导向，意思是绘画动作都依赖于当前的属性。

  • Current Transform Matrix
